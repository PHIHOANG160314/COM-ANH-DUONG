# /binh-phap - Strategic Planning Workflow

This workflow creates or updates the comprehensive strategic plan for the project.

## Steps

### 1. Analyze Current State
Review the current project structure:
- Check existing features and capabilities
- Review documentation (README, CICD_SETUP, etc.)
- Assess database schema and integrations
- Note any pending issues or bugs

### 2. Create/Update Strategy Document
The strategy document should cover:

#### Business Strategy
- KPIs and targets (customers, orders, ratings)
- Brand positioning and USP
- Customer segments

#### Technical Roadmap
- Phased feature development
- Architecture improvements
- Tech stack additions

#### Deployment Strategy
- Environment setup (prod, staging, preview)
- CI/CD pipeline health
- Domain and SSL status

#### Marketing & Operations
- Marketing channels and frequency
- Loyalty programs and tiers
- Campaign planning
- Daily operations SOP

- Present plan to stakeholders
